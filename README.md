grade your **Python** projects with **flake8** & **mypy**, and sit randomized **practice exams** —
all from one beautiful menu.

## 💡 Why 42-cli

If you're a **42 student** looking for a **42 tester** that *just works*, this is it.
Point it at a project folder and it auto-detects what you're building, runs the right
tests, checks the norm, and tells you — clearly — what passes and what doesn't.
And when you think you're ready, **Exam Practice** drops you into real exam conditions
so you don't get surprised by the moulinette.

<br/>

## ✨ Features

- 🧪 **Bundled testers** — libft (all 42 functions, or pick specific ones), ft_printf, get_next_line (multiple `BUFFER_SIZE`s).
- 📏 **Norminette** — run the norm and combined norm + tests in one go.
- ✅ **Subject compliance** — checks your Makefile, headers and required files match the subject.
- 🐍 **Python graders** — behavioural tests for the seven OOP / data-engineering projects, plus **flake8** (style) and **mypy** (type hints).
- 📝 **Exam Practice** — randomized, auto-graded mock exams in **C and Python**, with selectable count and difficulty (1–4, ascending or fixed).
- 🧭 **Module navigator** — browse the cursus with a clean tab bar (`← →` between modules, `↑ ↓` inside).
- 🔍 **Auto-detection** — run it from inside a project folder and it figures out which project you're on.
- 🏆 **Achievements** — 50+ unlockables to keep the grind fun.
- 🌍 **Bilingual** — full **English** and **Français** UI.
- 💌 **In-CLI feedback** — found a bug or a false pass? Send it to the maintainer without leaving the terminal.
- 🔄 **Self-updating** — `42 update` pulls the latest version.

## 🕹️ Quick start

```sh
42
```

That's it. Navigate the menu, pick a module, pick a project. From inside a `libft`
(or `ft_printf`, `get_next_line`, …) directory the CLI auto-detects the project, so
you can also just `cd` into your repo and run `42`.

```sh
42 --version     # print the installed version
42 update        # upgrade to the latest release
42 --help        # show all commands
```

## 📝 Exam Practice

Each active module has its own **Exam Practice** — C for Programming Fundamentals,
Python for everything after. It works like the real thing:

1. **Choose** how many exercises (1–5) and the difficulty: **ascending** (easy → hard)
   or a **fixed level** (1–4).
2. The CLI picks random exercises and scaffolds a fresh `exam-<timestamp>/` folder,
   one subject per exercise.
3. **Solve them** in your editor.
4. **Grade** each one: C solutions are compiled with `-Wall -Wextra -Werror` and run
   against hidden test cases; Python solutions are executed and checked — and you can
   run **flake8** and **mypy** on them too.

No spoilers, no peeking — just like exam mode. 🔒

## ❓ FAQ

<details>
<summary><b>Is this an official 42 tool?</b></summary>
No. It's a community-made, unofficial tester built by a 42 student to help other students. Always trust the official moulinette as the source of truth.
</details>

<details>
<summary><b>Does passing here mean I'll pass the moulinette?</b></summary>
It's a strong signal, not a guarantee. 42-cli mirrors the subjects as closely as possible, but only the official evaluation is final. If you pass here and fail there, please send feedback so the test can be fixed.
</details>

<details>
<summary><b>Which projects are supported right now?</b></summary>
libft, ft_printf, get_next_line and push_swap for C; the seven Python OOP / data-engineering projects; plus C & Python Exam Practice. More modules are on the roadmap.
</details>
